# 个股调研分析技能

## 适用场景
- 用户要求调研/分析某只A股或港股
- 用户要求看某公司的投资逻辑
- 用户要求判断某股票估值是否合理
- 用户提到"给我讲讲XX公司"、"XX怎么样"、"XX值不值得买"

## 输出物
结构化DOCX调研报告，包含：公司概况 → 产业布局 → 行业地位 → 财务数据 → 估值分析 → 情绪面分析 → 核心看点与风险 → 综合评价

## 执行步骤

### Step 1：确认标的
- 确认股票代码（如600667.SH / 002512.SZ）
- 确认公司全称和简称

### Step 2：搜索基础信息（并行执行）
同时发起以下搜索：
1. `{公司名} {代码} 主营业务 业务结构 产业布局 2025 2026`
2. `{公司名} {代码} 年报 营收 净利润 财务数据 2025`
3. `{公司名} {代码} 行业地位 竞争对手 市场份额 龙头`
4. `{公司名} {代码} 市盈率 市净率 估值 市值 最新`

### Step 3：搜索估值对标（并行执行）
1. 确定2-4家同行业可比公司
2. 搜索可比公司最新PE/PB/市值
3. 搜索券商目标价/评级
4. 搜索股票近期涨跌、资金流向、龙虎榜

### Step 4：搜索情绪面数据
1. `{公司名} 近期 涨停 原因 利好 催化 2026`
2. `{公司名} 资金流向 龙虎榜 主力`
3. `{公司名} 概念 板块 主题 炒作`

### Step 5：数据核实
- 财务数据以年报/季报公告为准，不用券商预测值替代实际值
- PE计算方式：总市值 / 最近四个季度归母净利润（TTM）
- 可比公司PE数据必须用同期最新数据，不能用历史值
- 如果搜索到的数据相互矛盾，以东方财富/同花顺/中财网等主流源为准

### Step 6：生成DOCX报告
使用docx skill的docx-js方式生成，报告结构如下：

```
封面：个股调研报告 + 公司名称代码 + 一句话定位 + 报告日期
一、公司概况（代码/上市时间/实控人/核心子公司/市值/股价）
二、产业布局与业务结构（分业务详述+收入占比+业务特征）
三、行业地位与竞争格局（龙头/二线/竞争对手表格对比）
四、核心财务数据（近期业绩表格+收入结构表格+关键指标）
五、估值分析（当前估值表格+同行对比表格+合理性判断）
六、情绪面与资金面分析（近期异动+资金信号+驱动逻辑+情绪风险）
七、核心看点与风险（看多逻辑+看空逻辑，各5-7条）
八、综合评价（300字以内的定性判断）
免责声明+数据来源
```

### Step 7：输出与展示
- 文件保存到 outputs/ 目录
- 使用 present_files 展示给用户
- 简短口头总结核心结论（3句话以内）

## 数据源优先级
1. 公司年报/季报公告（最权威）
2. 东方财富/同花顺/中财网实时数据
3. 财联社/证券时报/21世纪经济报道（行业报道）
4. 券商研报（观点参考，注意时效性）
5. 雪球/股吧（情绪参考，不作事实依据）

## 估值分析方法论
1. **绝对估值**：当前PE/PB vs 公司历史中枢 → 判断贵/便宜
2. **相对估值**：当前PE/PB vs 同行业可比公司 → 判断溢价/折价
3. **机构观点**：券商目标价 vs 当前股价 → 判断市场共识
4. **情绪溢价**：基本面能否支撑当前市值 → 判断情绪驱动程度
5. **板块环境**：同板块其他公司估值水平 → 判断是个股问题还是板块普涨

## 报告风格要求
- 客观呈现数据，不做买卖建议
- 看多看空逻辑并列，让读者自己判断
- 明确标注数据来源和时效
- 对争议性数据或不一致之处如实说明
- 情绪面分析独立成章，不与基本面混淆

## 常见坑
- PE数据来源不同口径差异大（TTM vs 静态 vs 动态），必须标注口径
- 券商目标价可能过时（3-6个月前的研报），需注意时效
- 不要用预测利润算PE，只用已发布的实际利润
- moomoo.com WebFetch经常超时，备选用亿牛网(eniu.com)、中财网等
- 半导体等高成长板块PE普遍较高，不能简单用传统行业标准衡量
- 情绪面和基本面要分开讨论，不能混为一谈

## 验证清单
- [ ] 所有财务数据有明确来源（年报/季报/公告）
- [ ] PE/PB计算口径已标注
- [ ] 可比公司数据为同期最新
- [ ] 情绪面有具体事件和数据支撑（龙虎榜/资金流/涨停原因）
- [ ] 看多看空逻辑平衡，无明显偏向
- [ ] DOCX格式无乱码，表格对齐
